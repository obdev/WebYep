Basics

The underlying intent of WebYep is to make parts of a web page editable for a "user":-  Someone accessing the page using a web browser, who provided the right username/password. You as the web designer only need to insert the appropriate WebYep element in place of a static text or image in the page and install the "webyep-system" folder in your website.

Quick setup guide:

  1. Copy the "webyep-system" folder contained within the WebYep package, into the root folder of your website (where your homepage - index.html - resides).
  2. Set the username and password in the file config-inc.php inside the "webyep-system" folder. You can edit the config file by opening it via a text editor. If you intend to edit the config file with Dreamweaver then switch to Code View after opening (this file is PHP code only!) and save it. If you are using Freeway you can edit and create the config file via the WebYep-Config Folder action.
  3. Upload the whole "webyep-system" folder to your webspace (via FTP).
  4. Prepare WebYep (see What does "Preparation" mean?

Creating pages - a quick guide:

  1. Create a normal HTML page but use the ".php" as the file's extension, not ".html".
  2. Insert any static (non-editable) content to design the page.
  3. In place of text or images that should be editable, add the appropriate WebYep Elements.

  5. Save and upload your page to the webspace (via FTP).
  6. Open the uploaded page up in your web browser.

Technical details

What does "Preparing" mean?

To see how to prepare WebYep using various applications, see "Installation".

What are "WepYep Elements"?

WebYep Elements are little fragments of PHP code inserted into the HTML code.

Layout of the webyep-system folder

  • webyep-system
  • config-inc.php (this is where you set up the username and password)
  • info.php (shows system information, installed licenses and WebYep status)
  • the program contains all WebYep program scripts and data
  • data (includes the data entered/uploaded by the users)
    • documents (list of pages using WebYep)
    • various files containing all data entered by the users (texts, images, etc.)

The "data" folder should be backed up from time to time (see "Backup").
